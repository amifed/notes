# 试除法判定质数

bool is_prime(int x) {
  if (x < 2) return 0;
  for (int i = 2; i <= x / i; ++i)
    if (x % i == 0) return 0;
  return 1;
}

# 朴素筛法求质数

从 2∼n2 \sim n2∼n 枚举，将其倍数删除，剩下的数就是质数

const int N = 1e6 + 7;
int ps[N], cnt;
bool vt[N];

void get_primes(int n) {
    for (int i = 2; i <= n; ++i) {
        if (!vt[i]) ps[cnt++] = i;
        for (int j = i + i; j <= n; j += i) vt[j] = 1;
    }
}

# 埃式筛法求质数

考虑这样一个事实：如果 xxx 是质数，那么大于 xxx 的 xxx 的倍数 2x,3x,…2x,3x,\ldots2x,3x,… 一定不是质数。

朴素筛法可做以下优化, 只将 2∼n2 \sim n2∼n 中的质数的倍数筛除即可。

这里还可以继续优化，对于一个质数 xxx，如果按上文说的我们从 2x2x2x 开始标记其实是冗余的，应该直接从 x⋅xx\cdot xx⋅x 开始标记，因为 2x,3x,…2x,3x,\ldots2x,3x,… 这些数一定在 xxx 之前就被其他数的倍数标记过了，例如 222 的所有倍数，333 的所有倍数等。

void get_primes(int n) {
    for (int i = 2; i <= n; ++i) {
        if (!vt[i]) {
            ps[cnt++] = i;
            for (int j = i * i; j <= n; j += i) vt[j] = 1;
        }
    }
}

# 线性筛法求质数

埃氏筛其实还是存在冗余的标记操作，比如对于 454545 这个数，它会同时被 3,53,53,5 两个数标记为合数，因此我们优化的目标是让每个合数只被标记一次，这样时间复杂度即能保证为 O(n)\mathcal{O(n)}O(n)。

循环中任何一个数 iii 只会被它的最小质因子筛掉，每个数都会有一个最小质因子，因此每个数都只会被筛一次。

内层 for\textit{for}for 循环是从头开始遍历质数表中的质数，无论 iii 为质数或合数，都会枚举到它的最小质因子

  1. i%ps[j]=0i\;\%\;ps[j] = 0i%ps[j]=0 时，ps[j]ps[j]ps[j] 为 iii 的最小质因子，ps[j]ps[j]ps[j] 一定是 ps[j]×ips[j] \times ips[j]×i 的最小质因子；
  2. i%ps[j]≠0i \ \% \ ps[j] \ne 0i%ps[j]=0 时，ps[j]ps[j]ps[j] 一定小于 iii 的所有质因子，ps[j]ps[j]ps[j] 也一定是 ps[j]×ips[j] \times ips[j]×i 的最小质因子。
const int N = 1e6 + 7;
int ps[N], cnt;
bool vt[N];

void get_primes(int n) {
    for (int i = 2; i <= n; ++i) {
        if (!vt[i]) ps[cnt++] = i;
        for (int j = 0; ps[j] <= n / i; ++j) { //从头开始遍历质数表中的质数
          vt[ps[j] * i] = 1;
          if (i % ps[j] == 0) break;
        }
    }
}
